# VISION.md テンプレ

`loop-engineering` スキルの STEP2 で使う「合格条件書」の雛形。そのままコピーして埋める。

```markdown
# VISION — <機能/修正の名前>

## ゴール(1〜2行・人間向け)
<このタスクが達成された世界を一言で>

## 完了条件(Stop condition)
- [ ] `npm run verify`(typecheck && lint && test 相当)の exit code が 0
      ※ まとめコマンドが無い場合: typecheck / lint / test を個別実行し全部 exit 0
- [ ] VISION の全 [機械] 条件に対応するテストが存在し、緑である(同一 ID で照合)
- [ ] reviewer が `NO_ISSUES`(CRITICAL/HIGH の指摘ゼロ)
- [ ] (`[AI]` 条件がある場合のみ)AI 検証がすべて pass
- [ ] 往復は最大 5 回。超えたら止めて人間に相談

## 合格条件(=テスト観点=テスト。すべて同じ ID で貫通)
| ID | 検証手段 | 条件(○○すると △△ になる) |
|----|----------|------------------------------|
| N1 | [機械]   | <正常系> |
| E1 | [機械]   | <異常入力 → 期待されるエラー> |
| B1 | [機械]   | <境界値ちょうど / +1 の挙動> |
| S1 | [機械]   | <状態遷移・連続操作・永続化> |
| Q1 | [AI]     | <見た目/文言など曖昧な品質。無ければ省略> |

## スコープ外(やらないこと)
- <今回触らないもの。スコープ膨張を防ぐ>
```
