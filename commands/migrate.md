# マイグレーション実行

**migration-runner エージェント** を起動して、DB スキーマの変更を適用します。

## このコマンドが行うこと

1. **マイグレーションツールの検出** — Alembic・Prisma・Laravel・Rails・Drizzle 等
2. **ステータス確認** — 未適用マイグレーションのリスト表示
3. **Dry-run** — 実行される SQL を事前表示（破壊的変更は確認）
4. **バックアップ確認** — 本番環境では事前にバックアップを確認
5. **マイグレーション実行** — ツール固有のコマンドを実行
6. **検証** — 未適用が無くなったか・アプリが起動するか確認
7. **結果レポート** — 適用したマイグレーション・所要時間

## いつ使うか

- スキーマ変更を伴う PR をマージした後
- デプロイ前に DB 変更が必要なとき（`/deploy` の前段）
- スキーマを本番に追従させたいとき

## 前提

- プロジェクトにマイグレーション設定が存在する（Alembic・Prisma 等）
- DB 接続情報が環境変数 / 設定ファイルに揃っている

## 関連コマンド

- `/deploy` — マイグレーション後のデプロイ実行
- `/rollback` — DB を含むロールバックが必要なとき

## 関連エージェント

`~/.claude/agents/migration-runner.md` を起動します。

## 注意事項

⚠️ **本番マイグレーションは必ずユーザー承認を求めます。**
⚠️ **破壊的変更（DROP・RENAME など）は dry-run の SQL を提示してから実行します。**
