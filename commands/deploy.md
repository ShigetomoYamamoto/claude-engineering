# デプロイ

**deploy-runner エージェント** を起動して、現在のコードベースをデプロイします。

## このコマンドが行うこと

1. **デプロイ設定の検出** — `.github/workflows/`・`vercel.json`・`fly.toml` 等から判別
2. **事前チェック** — ビルド成功・テスト PASS・マイグレーション準備・環境変数
3. **デプロイ実行** — プロジェクト固有のコマンドを実行
4. **検証** — HTTP ヘルスチェック・スモークテスト
5. **自動ロールバック** — 検証失敗時に前バージョンへ戻す
6. **結果レポート** — デプロイ URL・所要時間・検証結果

## いつ使うか

- 全自動モードの最終ステップ（PR マージ後）
- 手動でデプロイを実行したいとき

## 前提

- プロジェクト側にデプロイ設定が存在する（または手動で指定する）
- 環境変数・シークレットが設定済み
- マイグレーションが必要な場合は事前に `/migrate` を実行

## 関連コマンド

- `/migrate` — デプロイ前のマイグレーション実行
- `/rollback` — 後から手動でロールバックしたいとき

## 関連エージェント

`~/.claude/agents/deploy-runner.md` を起動します。

## 注意事項

⚠️ **本番デプロイは事前チェックに警告がある場合、ユーザー確認を求めます。**
