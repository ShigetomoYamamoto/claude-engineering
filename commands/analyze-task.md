# タスク分析

**task-analyst エージェント**を起動して、渡されたタスク・Issue・バグ報告を実装可能な単位に分解します。
分類・抽出・分解・承認待ちの手順はすべて **エージェント本体が単一の正**(このコマンドは薄い起動器。ADR-014 / 監査 P5)。

## いつ使うか

- サポートモードの起点として(タスク → PR)
- GitHub Issue・バグ報告・改善依頼を受けたとき
- 入力が**具体的なタスク/Issue**なら `/analyze-task`、**曖昧な要望/新機能アイデア**なら `/requirements`

## 次のステップ

承認後: `/plan`(実装計画)→ 実装は `loop-engineering`(「実装して」で起動)。

## 関連エージェント

`~/.claude/agents/task-analyst.md` を起動します(分析をユーザーが承認するまで実装に進みません — 詳細はエージェント本体)。
