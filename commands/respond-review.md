# PR レビュー対応

**review-responder エージェント** を起動して、PR にもらったレビューコメントに対応します。

## このコマンドが行うこと

1. **PR 状態の取得** — `gh pr view` でレビューコメント一覧を取得
2. **コメントの分類** — 必須修正 / 提案 / 質問 / 些細な指摘 / 賞賛
3. **修正の実装** — 必須修正と妥当な提案を実装
4. **返信の投稿** — 「<コミットハッシュ> で対応」「<理由> により据え置き」など
5. **コミット & プッシュ** — 関連する修正をまとめてコミットし、PR ブランチへプッシュ
6. **スレッドのクローズ** — 妥当な範囲で解決済みマーク

## いつ使うか

- PR にレビューがついて修正が必要になったとき
- サポートモードの後半（PR 作成後の対応）

## コードレビュー（公式 `pr-review-toolkit`）との違い

| コマンド / エージェント | 役割 |
|---|---|
| `code-reviewer`（公式 `pr-review-toolkit`） | 自分のコードをレビュー（問題を探す） |
| `/respond-review` | 他者のレビューに対応（指摘された問題を直す） |

## 関連エージェント

`~/.claude/agents/review-responder.md` を起動します。

## 注意事項

⚠️ **force push が必要な場合はユーザー確認を求めます。**
⚠️ **ユーザーが同意しないスレッドを勝手に解決済みにしません。**
⚠️ **レビューコメントごとに「何をやったか」を具体的に返信します。**
