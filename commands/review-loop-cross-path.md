別モデル(Codex)が **指定パス `<path>` を丸ごと**レビューする review-loop。git 差分ではなくパス指定なので **非 git・差分が無い状況でも使える**。修正は Claude の `fixer` が担当する。

> ⚠️ `<path>` は**必須**。
> ⚠️ 既定上限は **2**(差分ではなく全体レビューは指摘が出やすいため、他コマンドの 5 と異なる)。5 回回したいなら第2引数で明示的に渡す。
> ⚠️ 外部 CLI(codex)実行=ネットワーク通信・外部サービス利用量(課金)の副作用。起動時に利用者へ確認。

# 前提確認(ループ開始前に1回)

1. **`<path>` の指定確認**: 渡されていなければ「レビュー対象のパスを指定してください(例: `/review-loop-cross-path src/`)」と案内して停止。
2. **codex CLI の存在確認**: `command -v codex`。無ければ案内して停止。
3. **利用者への確認**: 外部モデル(課金・通信)使用の承認を得てから開始。

# 初回セットアップ

親が以下を保持する(`review-loop-cross` と同型。違いは SCOPE がパス指定な点):

- **SCOPE**: `<path>`(ファイル/ディレクトリ)。固定。
- **FOCUS**: 重点観点(任意)。
- **REVIEW_HISTORY / FIX_HISTORY**: 空配列で初期化。

# 実行ループ

`$ARGUMENTS` の第2引数(既定 **2**)を **MAX**、カウンタを **N** とする。

## ステップ1: レビュー(別モデル・パス指定)

```bash
codex exec "あなたは厳格なシニアコードレビュアー。次のパス配下を読んでレビューし、CRITICAL/HIGH の問題のみを『[重大度] [ファイル:行] 要約 / 影響 / 修正方針』形式で列挙せよ。HIGH 以上が無ければ NO_ISSUES の1行のみ。MEDIUM/LOW は出力しない。対象パス: <path>"
```

> 出力契約は `~/.claude/agents/reviewer.md` と同一に正規化する(CRITICAL/HIGH のみ・無ければ `NO_ISSUES`)。

出力を `CURRENT_REVIEW` として保持。

## ステップ2〜6

`review-loop-cross` と同じ(終了判定 → 重複検知 → `fixer` で修正 → 進捗報告 → 継続)。修正対象は SCOPE(`<path>`)配下に限定する。

# 終了条件・最終報告

`/review-loop` と同じ。報告に **「レビュー担当: 外部モデル(codex) / 対象: `<path>`」** を明記。

# 注意

- 既定上限 **2** を黙って増やさない(`~/.claude/rules/loop-safety.md` のハードストップ準拠。引数で明示された場合のみ変更)。
- **レビュー=外部モデル、修正=Claude の fixer** の分業を厳守。
- 安全規律の正本は `~/.claude/rules/loop-safety.md`。
